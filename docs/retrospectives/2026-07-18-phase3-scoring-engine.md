## 3. 패널이 잡아낸 핵심 결함 (수정 전 → 수정 후)

패널은 7개 코드 태스크 중 6개에서 Critical/Important를 발견해 수정시켰다
(T4만 첫 라운드 전원 승인). 대표 사례:

1. **T7 트레이더 Critical — 자정 스코어링 상시 실패.** 신선도 게이트 기준일이
   `previous_weekday(now)`(오늘)여서 자정 실행 시 전날 밤 수집분이 전부 낡음
   판정 → 스펙이 명시한 정규 실행이 매번 실패하는 스펙 자체 결함(코디네이터가
   의심 제기, 트레이더가 확정). → `scoring_reference_date`("오늘 이전의 마지막
   평일") 신설·기본 배선. 모든 기존 테스트가 기준일을 주입해서 이 기본값 결함을
   못 잡았다는 교훈 → 주입 없는 기본값 회귀 테스트 추가.
2. **T1 트레이더 — 은행 섹터 소실.** 021(금융)을 우산 제외하면 어느 하위
   코드에도 없는 98종목(은행·지주)이 로테이션에서 사라짐 — ka20002 집합 실측
   (021 ⊇ 024∪025 완전 포함)으로 확정 후 021을 industry로 승격. **T8 실데이터
   에서 021이 선정 업종이 됐고 후보 #1·#3(신한지주 포함)을 배출** — 수정의
   가치가 실측으로 입증됨.
3. **T6 트레이더 — 표본 부족 게이트가 정규화로 무력화.** occurrences<3 → 원점수
   0이지만 min-max 상대 정규화에서 0이 음수 성과보다 위로 올라가는 역전(전량
   게이트 시 균일 0.5) → 정규화 이후 0 클램프 + 판별 테스트.
4. **T6 개발자 — 점수 재현 불가.** 저장된 strategy_score가 total_score 계산에
   실제 쓰인 정규화값과 달라 공식이 저장값으로 재현 안 됨(계획서 설계 결함) →
   `strategy_score_norm` 칼럼 신설, isclose 왕복 테스트.
5. **T2 트레이더 Critical — 중간 상태 크래시.** `upsert_sectors` 필수 인자
   추가로 T2 커밋 단독 노출 시 야간 수집 전체가 TypeError로 중단 → 옵셔널
   폴백으로 중간 상태 자체를 안전화.
6. **T3 트레이더 — 업종 실패 격리 부재.** 65그룹 중 1개의 조회 실패가 봉 수집
   까지 통째로 중단 → 업종별 격리 + **부분 실패 시 전체 교체 스킵**(실패 업종의
   기존 소속이 삭제되지 않도록 직전 완전 스냅샷 보존 — "낡았지만 완전 > 최신이지만
   구멍" 판단).
7. **T5 개발자 — 판별력 없는 테스트.** 눌림목 밴드 테스트가 단락 평가로 밴드
   조건에 도달하지 못함(계획서 손계산 오류) → 다른 조건 전부 참 고정 + 밴드만
   스위칭하는 판별 쌍 테스트.
8. **T7 아키텍트 — 상호 배제가 API에만.** Phase 6 스케줄러가 HTTP를 우회하면
   뚫리는 구조 → conflict_check를 도메인 계약으로 이동 + `_running` 영구 고착
   결함(형제 서비스 복제 포함) 수정.
