## 0. 무엇을 만들었나 (비전문가용 요약)

키움 REST에는 서버측 손절/익절 예약 기능이 없다(§5 실측 팩트). 그래서
Phase 5는 **클라이언트가 직접** 신호 종목에 진입하고, 1초 주기로 시세를
관측해 손절(-5%)·트레일링·익절 문턱을 판정하고, 시장가로 청산하는 엔진
전체를 구현했다. 자금 사고를 막는 버그 봉쇄 한도(TRADE_* 4종 하드
게이트), 킬스위치 2모드, 크래시 후 재기동 복구(reconcile), 전 주문·체결·
포지션의 감사 영속이 함께 들어갔다.

**최종 검증 상태**: 진입→감시→손절→청산→크래시 복구의 전 수명주기가
실측 시장 데이터 위에서 end-to-end로 증명됨(리플레이 R7 + 실서버 부분 —
§10 참조).

  트레일링 2단 폭 **선형 보간**(계단식은 전환점 휘핑쏘 — 트레이더 v3),

## 3. 프로세스 회고

- **패널 리뷰 유효성**: 전 태스크에서 Critical/Important가 나왔고 대부분
  실질 결함이었다. 특히 인상적인 것: bp 부동소수점 경계(개발자, 실측
  재현), 매도 무차단 원칙(트레이더 — 안전장치의 역설), 공휴일 누락
  (트레이더 교차 검증), tz 미정규화(아키텍트 — 타 태스크 리뷰 중 발견).
- **실측이 문서를 이긴 횟수**: 파이프 구분자, trde_tp 자릿수, 취소 필드
  명, io_tp_nm 형태, 틱 검증 신호 — 전부 공식/사제 문서와 실측이 달랐고
  실측을 채택했다(규칙 8-b broker-api 패널 신설의 근거).
- **검증 환경의 자기 검증**: Task 8 라이브가 막히자(픽 0 + 갭) 리플레이
  가 그 공백을 메웠고, 역으로 리플레이 예행이 게이트 정의 불일치·시계
  혼합 같은 **검증 도구 자체의 결함**을 찾아냈다(R7 발견 2건 — 당일
  패치·패널·커밋 `4d378be`). 검증기가 검증되는 구조가 됐다.
- **환경 이전 사건**: Windows Docker Desktop→리눅스 네이티브 서버 이전이
  뒤늦게 발견됨(빈 DB·host.docker.internal 부재·Ollama 미설치). 야간
  풀 수집 재구축+compose host-gateway+Ollama 브리지 바인딩(ufw 대역
  제한)으로 해소. **교훈: 환경 전제(가상화 종류·데몬 존재)는 STATUS에
  명시할 것.**
